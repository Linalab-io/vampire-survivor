## 1. 기획 단계 — 팀원들의 소크라테스식 질의응답

### 1.1 시작 (2026-05-06)

"뱀파이어 서바이버 라이크 장르의 게임을 pyxel 로 만들거야 계획을 세워줘"

이 한마디로 시작된 프로젝트. Prometheus(계획 에이전트)가 소크라테스식 질문을 통해 모호성 레벨을 0.90에서 0.03까지 낮추며, 15명의 팀원이 돌아가며 하나씩 질문에 답했습니다.

> "질문은 한번에 하나씩만 해줘" — 이 규칙 하나로, 한 사람이 한 질문에 답하는 릴레이 방식이 만들어졌습니다.

## 2. 구현 단계 — MVP (Task 1-36)

Sisyphus(오케스트레이터)가 Prometheus의 계획을 기반으로 36개 태스크를 8개 Phase로 나누어 Sisyphus-Junior(deep 카테고리)에게 위임 실행했습니다.

### Phase 1: 기반 구조 (Task 1-5)

- **Task 1**: Pyxel 앱 스켈레톤 — `pyxel.init(160, 120)`, App 클래스, update/draw 루프
- **Task 2**: 상태머신 — if/elif 분기, 상태 전환 로직
- **Task 3**: 스프라이트 정의 — SPR_KNIGHT, SPR_SKELETON 등 16×16 픽셀 아트
- **Task 4**: 타이틀 화면 — 테두리 장식, 깜빡이는 "PRESS ENTER"
- **Task 5**: HUD — HP 바(빨강), XP 바(시안), 레벨 텍스트, MM:SS 타이머

### Phase 2: 플레이어 시스템 (Task 6-9)

- **Task 6**: 8방향 이동 — Arrow + WASD, 대각선 0.707 정규화 (오수빈·박한나의 결정 반영)
- **Task 7**: 무한 스크롤링 맵 — `_tile_hash(wx, wy)` 결정론적 타일 (김용수의 결정 반영)
- **Task 8**: 대시 능력 — 40px 순간이동, 60프레임 쿨다운 (이준호의 결정 반영)
- **Task 9**: 바이옴 전환 — 30초 전환, Grassland→Desert→Cave→Castle (박한나의 환경 타일 결정 반영)

### Phase 3: 전투 시스템 (Task 10-14)

- **Task 10**: 적 스폰 — 50마리 캡 (이동민의 결정 반영), 112px 스폰 반경
- **Task 11**: Skeleton AI — 직선 추적, 12px 접촉 데미지, 60프레임 무적 창
- **Task 12**: Whip 무기 — 좌/우 교대 공격 (조재국의 채찍 결정 반영), 45프레임 쿨다운
- **Task 13**: 히트박스/데미지 — rect_overlap 충돌, 넉백 15px
- **Task 14**: XP 젬 — 적 사망 시 드롭, 자석 범위 30px (이상용의 자석 결정 반영)
