# Vampire Survivors Pyxel — Evolution Roadmap

> **From**: 완성된 VS 클론 MVP (main.py 2683줄, 36 태스크 + 4 검증 모두 PASS)
> **To**: 폴리시 + 리팩토링 + 메타 프로그레션이 추가된 진화 버전
> **Date**: 2026-05-07

---

## Overview

MVP가 완성된 상태에서 3가지 방향으로 진화:

1. **Phase A**: 버그 수정 + 폴리시 — 알려진 이슈 해결, 시각/오디오 퀄리티 업
2. **Phase B**: 리팩토링 + 기술부채 — 싱글 파일 구조 개선, 성능 최적화
3. **Phase C**: 메타 프로그레션 — 언록 시스템, 영구 업그레이드, 업적

### Scope

**포함 (Must Have)**
- Phase A: 6개 알려진 이슈 전부 수정
- Phase A: 화면 흔들림, 파티클, 피격 피드백 등 시각 폴리시
- Phase A: 더 많은 SFX + 기본 BGM 루프
- Phase B: main.py → 모듈 분리 (최대 8개 파일)
- Phase B: 공간 해시 기반 충돌 최적화
- Phase B: 설정 파일 (키 바인딩, 볼륨)
- Phase C: 런 종료 후 코인 획득 → 영구 업그레이드 상점
- Phase C: 캐릭터/무기 언록 시스템
- Phase C: 업적 + 통계 추적 (JSON 세이브)
- Phase C: 아케이드 모드 (하이스코어)

**미포함 (Must NOT Have)**
- ❌ 온라인/멀티플레이어
- ❌ 하이레스 외부 에셋 (Pyxel 16색 유지)
- ❌ Unity 의존성
- ❌ 과도한 추상화 (엔티티 컴포넌트 시스템 등)
- ❌ 스토리/대화 시스템
- ❌ 타일맵 에디터

## Phase A: 버그 수정 + 폴리시 (13 tasks)

### A1. 캐릭터 스프라이트 렌더링 수정 ★MAJOR
- **문제**: 모든 캐릭터가 SPR_KNIGHT로 렌더링됨
- **해결**: CHARACTER_DATA에서 sprite index/color를 참조하여 draw_playing에서 적용
- **검증**: 캐릭터 선택 → 게임 플레이 시 선택한 캐릭터 색상 표시
- [x] A1: 캐릭터 스프라이트 렌더링 수정

### A2. 하드코딩 FPS 상수화
- **문제**: 타이머 계산에 `30` 하드코딩 (6곳)
- **해결**: 모두 `FPS` 상수로 교체
- **검증**: grep으로 `// 30` 또는 `/ 30` 패턴 잔여 확인
- [x] A2: 하드코딩 FPS 상수화

### A3. 이중 cls() 오버드로우 제거
- **문제**: 공통 draw()와 각 상태별 draw에서 cls() 중복 호출
- **해결**: 공통 cls()만 유지, 상태별 cls() 제거
- **검증**: cls() 호출이 정확히 1곳만 존재
- [x] A3: 이중 cls() 오버드로우 제거

### A4. Clover/Luck 패시브 효과 활성화
- **문제**: luck 스탯이 정의만 되고 실제 효과 없음
- **해결**: luck → 크리티컬 확률 또는 XP 젬 드롭률에 적용
- **검증**: Clover 장착 시 luck 수치 변화 → 효과 확인
- [x] A4: Clover/Luck 패시브 효과 활성화

### A5. 디버그 키 게이트 추가
- **문제**: F1-F4가 프로덕션에서도 항상 활성화
- **해결**: 타이틀 화면에서 특정 키 조합으로 unlock 또는 환경변수 게이트
- **검증**: 기본 상태에서 F1-F4 미동작, unlock 후 동작
- [x] A5: 디버그 키 게이트 추가

### A6. 일시정지 컨트롤 안내문 수정
- **문제**: X 키 대시 표기 누락
- **해결**: "ESC: 계속 | X: 대시" 등 전체 컨트롤 표시
- **검증**: 일시정지 화면 스크린샷
- [x] A6: 일시정지 컨트롤 안내문 수정

### A7. 화면 흔들림 (Screen Shake)
- **범위**: 피격 시 2-3프레임 화면 오프셋, 보스 등장 시 강한 흔들림
- **구현**: draw 시 camera_x/y에 shake_offset 추가, 매 프레임 감쇠
- **검증**: 적에게 피격 → 화면 흔들림 관찰

- [x] A7: 화면 흔들림 (Screen Shake)

### A8. 파티클 시스템
- **범위**: 적 처치 시 이펙트, 레벨업 이펙트, XP 젬 수집 이펙트
- **구현**: 간단한 particle_list + color/size/lifetime/velocity
- **제한**: 동시 최대 30개 파티클
- **검증**: 적 처치 → 파티클 스폰 → 자연스럽게 소멸

- [x] A8: 파티클 시스템

### A9. 피격 피드백 (Hit Flash)
- **범위**: 적 피격 시 2프레임 흰색 플래시, 플레이어 피격 시 3프레임 빨간색 오버레이
- **검증**: Whip 공격 → 적 흰색 플래시 확인

- [x] A9: 피격 피드백 (Hit Flash)

### A10. 무기 레벨 표시 (HUD)
- **범위**: HUD 하단에 장착 무기 아이콘 + 레벨 표시
- **구현**: weapon_list 순회하여 sprite + Lv.N 텍스트
- **검증**: 무기 획득/업그레이드 시 HUD 반영

- [x] A10: 무기 레벨 표시 (HUD)

### A11. BGM 루프 추가
- **범위**: 게임플레이 중 반복되는 간단한 칩튠 루프 (pyxel.soundm에 등록)
- **구현**: 4채널 루프 패턴, 볼륨 제어 가능
- **제한**: 기존 SFX 채널과 충돌하지 않도록 채널 분리
- **검증**: 게임 시작 → BGM 재생 확인

### A12. 추가 SFX
- **범위**: 대시, 진화 성공, 보스 등장, 바이옴 전환, 메뉴 이동
- **검증**: 각 이벤트 발생 시 해당 사운드 재생

### A13. 페이드 전환 효과
- **범위**: 화면 전환 시 (타이틀→선택→게임→결과) 10프레임 페이드 인/아웃
